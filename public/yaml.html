Sintaxe Essencial e Tipos de Dados

O YAML se baseia em dois conceitos principais: pares chave-valor para representar dados e indentação (espaços) para denotar estrutura. Ele infere automaticamente os tipos de dados mais comuns.

Conceitos Fundamentais

O YAML é uma linguagem de serialização de dados legível por humanos. Sua simplicidade e clareza o tornam ideal para arquivos de configuração, especialmente em DevOps e desenvolvimento.

Exemplo de Sintaxe Básica

# Pares Chave-Valor
chave: valor

# Strings (com ou sem aspas)
nome_do_site: "Meu Site Incrível"
outro_nome: Apenas um site

# Números (inteiros e decimais)
idade: 35
preco: 19.99

# Booleanos (verdadeiro ou falso)
publicado: true
em_destaque: false

# Valor Nulo
descricao: null

Pontos-Chave

  • Chave-Valor: A base de tudo no YAML. A sintaxe é `chave: valor`. Note o espaço após os dois pontos.
  • Indentação é Estrutura: O YAML usa espaços (não tabs!) para criar aninhamento e hierarquia. A consistência no número de espaços é crucial.
  • Comentários: Qualquer texto após um `#` é um comentário e é ignorado.
  • Tipos de Dados: YAML reconhece automaticamente `strings`, `números`, `booleanos` (`true`/`false`) e `null` (nulo).

Estruturas de Dados: Listas e Dicionários

Para organizar dados, o YAML usa listas (sequências) e dicionários (mapeamentos). A combinação dessas duas estruturas permite modelar praticamente qualquer dado complexo.

Listas (Sequências)

Listas são coleções ordenadas de itens. No YAML, você pode criar listas usando hífens ou colchetes.

Sintaxe de Listas

# Uma lista simples de strings
linguagens:
  - Python
  - JavaScript
  - Go
  - Rust

# Lista inline (mais compacta)
cores: [vermelho, verde, azul]

# Lista de objetos
desenvolvedores:
  - nome: João
    idade: 28
    especialidade: Backend
  - nome: Maria
    idade: 32
    especialidade: Frontend

Dicionários (Mapeamentos)

# Dicionário simples
configuracao:
  banco_dados:
    host: localhost
    porta: 5432
    usuario: admin
    senha: secret123

# Dicionário inline
contato: {nome: "João Silva", email: "joao@email.com"}

# Estruturas aninhadas
projeto:
  nome: "Meu Projeto"
  versao: "1.0.0"
  dependencias:
    - express
    - mongoose
  scripts:
    start: "node server.js"
    test: "npm test"

Recursos Avançados

O YAML oferece recursos poderosos para reutilização de código, referências e formatação avançada que tornam os arquivos mais limpos e organizados.

Ancoras e Referências

Use âncoras (`&`) para definir valores reutilizáveis e referências (`*`) para referenciá-los. Isso evita duplicação e facilita a manutenção.

Exemplo de Âncoras

# Definindo âncoras
padroes: &padroes
  timeout: 30
  retries: 3
  debug: false

# Usando referências
desenvolvimento:
  <<: *padroes
  debug: true
  database_url: "localhost:5432"

producao:
  <<: *padroes
  database_url: "prod-server:5432"
  timeout: 60

Strings Multilinhas

# String literal (preserva quebras de linha)
descricao: |
  Esta é uma descrição
  que pode ter múltiplas
  linhas e será preservada
  exatamente como escrita.

# String dobrada (quebra linhas em espaços)
resumo: >
  Esta é uma descrição
  que será dobrada em uma
  única linha com espaços
  entre as palavras.

# String com preservação (mantém quebras de linha finais)
codigo: |+
  function hello() {
    console.log("Hello World");
  }

Exemplo Prático: Docker Compose

Veja como o YAML é usado na prática com um arquivo docker-compose real. Clique em diferentes partes do código para entender cada conceito.

Docker Compose Interativo

Este exemplo mostra um arquivo docker-compose.yml completo com múltiplos serviços, redes e volumes.

Arquivo docker-compose.yml

version: '3.8'

services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    environment:
      - NGINX_HOST=localhost
      - NGINX_PORT=80
    depends_on:
      - app

  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://user:pass@db:5432/mydb
    depends_on:
      - db

  db:
    image: postgres:13
    environment:
      - POSTGRES_DB=mydb
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

networks:
  default:
    driver: bridge
