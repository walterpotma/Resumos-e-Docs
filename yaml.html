Sintaxe Essencial e Tipos de Dados

O YAML se baseia em dois conceitos principais: pares chave-valor para representar dados e indentação (espaços) para denotar estrutura. Ele infere automaticamente os tipos de dados mais comuns.

# Pares Chave-Valor
chave: valor

# Strings (com ou sem aspas)
nome_do_site: "Meu Site Incrível"
outro_nome: Apenas um site

# Números (inteiros e decimais)
idade: 35
preco: 19.99

# Booleanos (verdadeiro ou falso)
publicado: true
em_destaque: false

# Valor Nulo
descricao: null

Pontos-Chave:

  • Chave-Valor: A base de tudo no YAML. A sintaxe é `chave: valor`. Note o espaço após os dois pontos.
  • Indentação é Estrutura: O YAML usa espaços (não tabs!) para criar aninhamento e hierarquia. A consistência no número de espaços é crucial.
  • Comentários: Qualquer texto após um `#` é um comentário e é ignorado.
  • Tipos de Dados: YAML reconhece automaticamente `strings`, `números`, `booleanos` (`true`/`false`) e `null` (nulo).

Estruturas de Dados: Listas e Dicionários

Para organizar dados, o YAML usa listas (sequências) e dicionários (mapeamentos). A combinação dessas duas estruturas permite modelar praticamente qualquer dado complexo.

Listas (Sequências)

# Uma lista simples de strings
linguagens:
  - Python
  - JavaScript
  - Go

Uma lista é uma coleção de itens. Cada item começa com um hífen e um espaço (`- `). Todos os itens devem ter o mesmo nível de indentação.

Dicionários (Mapeamentos) Aninhados

# Um dicionário com outro dicionário dentro
configuracao:
  database:
    host: localhost
    porta: 5432
    usuario: admin

Um dicionário (ou mapeamento) é uma coleção de pares chave-valor. Você pode aninhar dicionários para criar hierarquias, simplesmente indentando o dicionário interno.

Listas de Dicionários (Padrão Comum)

# Uma lista onde cada item é um dicionário
usuarios:
  - nome: Alice
    email: alice@email.com
    admin: true
  - nome: Bob
    email: bob@email.com
    admin: false

Este é um dos padrões mais poderosos e utilizados. `usuarios` é uma lista, e cada elemento (`-`) é um objeto complexo (um dicionário) com suas próprias chaves e valores.

Recursos Avançados e Melhores Práticas

O YAML possui recursos que ajudam a manter seus arquivos limpos, legíveis e sem repetições, especialmente em configurações complexas.

Strings de Múltiplas Linhas

Para textos longos, você pode usar `|` para preservar as quebras de linha (literal) ou `>` para dobrá-las em uma única linha (dobrado).

# Estilo Literal: Preserva quebras de linha
script_shell: |
  echo "Iniciando processo..."
  python main.py
  echo "Finalizado."

# Estilo Dobrado: Converte quebras de linha em espaços
paragrafo: >
  Este texto era muito longo e foi
  escrito em várias linhas, mas
  o YAML irá juntar tudo
  em uma única linha.
  • Use | (estilo literal) quando o espaçamento e as quebras de linha são importantes, como em scripts, poesia ou logs.
  • Use > (estilo dobrado) para descrições ou parágrafos longos, onde você quer que o resultado final seja uma string contínua, mas o código-fonte permaneça legível.

Âncoras (`&`) e Aliases (`*`) para Reutilização

Para evitar a repetição de blocos de código (princípio DRY), você pode definir uma âncora e reutilizá-la em outros lugares com um alias.

# Define uma âncora com um bloco de configuração
config_padrao_app: &config_app
  memoria: 256M
  restart: always

# Reutiliza a âncora nos serviços
servico_api:
  <<: *config_app # Mescla a âncora aqui
  nome: "API Principal"
  build: ./api

servico_worker:
  <<: *config_app # Mescla a âncora aqui
  nome: "Worker de Fila"
  build: ./worker
  • &nome_da_ancora define um bloco de dados reutilizável.
  • *nome_da_ancora insere o bloco de dados definido pela âncora.
  • <<: é uma chave especial de "mesclagem", que funde as chaves do alias no dicionário atual. É extremamente útil para herdar configurações padrão.

Exemplo Prático: `docker-compose.yml`

services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf

  database:
    image: postgres:14-alpine
    restart: always
    environment:
      POSTGRES_DB: meudb
      POSTGRES_USER: usuario
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data: {}
